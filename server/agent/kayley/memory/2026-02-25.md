## 🔬 Research TODO — 2026-02-26

### Supabase Duplicate Insert PR (openclaw/openclaw #25835)
- **Issue:** Gateway injects duplicate Telegram messages (~20s apart, same `message_id`) into session JSONL, causing LLM to respond twice → duplicate Supabase inserts
- **Their fix:** Deduplication at the gateway level by `message_id` before session injection
- **Our fix:** Downstream deduplication in `hooks/supabase-convo-log/handler.ts` log processor (matches the "workaround" described in the issue)
- **Action when PR merges:** Review our custom dedup logic in the hook handler and likely **remove it** to avoid redundancy/conflicts with the upstream fix
- **Link:** https://github.com/openclaw/openclaw/issues/25835
- **Status as of 2026-02-25:** PR open, not yet merged. Error logs for inserts are just noise for now.

### Model Switching / Allowlist Bug (openclaw/openclaw #20291)
- **Issue:** `/model` command + session-level model switching fails with "model not allowed" even when `agents.defaults.models` allowlist is correctly configured
- **Root cause:** Gateway uses a different code path for session-level overrides that doesn't read from `agents.defaults.models` — the allowlist check is effectively bypassed at runtime
- **We experienced this today:** Fallback chain was getting rejected even after adding models to the allowlist
- **Workaround in place:** Patching `agents.defaults.model.primary` via config + restart instead of runtime switching
- **Link:** https://github.com/openclaw/openclaw/issues/20291
- **Action:** Monitor for a fix/PR; when merged, test that fallback + model switching works natively without the config-patch workaround
