# Lessons Learned — f4357742-f318-455f-9941-6c5fa3cbf697 — 2026-03-07

## Ticket
Add time and status filtering for server runtime logs in the admin dashboard.

## Codebase Discoveries
- Runtime log filtering is handled entirely client-side in `src/services/adminService.ts` via Supabase queries; the admin dashboard just passes filter options.
- `server_runtime_logs` consistently writes `occurred_at`, so time filters can use that column directly without extra joins.

## Gotchas & Bugs
- Server logs do not use `source='server'`; filtering server logs requires `neq('source', 'client')`.

## Approach That Worked
- Added a time-window dropdown and translated it to a `since` ISO timestamp for the Supabase query.

## What Future Opey Should Know
- Keep runtime log filters lightweight; the admin service is the single place to add new query constraints.
