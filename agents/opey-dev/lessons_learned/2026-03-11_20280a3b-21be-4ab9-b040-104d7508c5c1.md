# Lessons Learned ? 20280a3b-21be-4ab9-b040-104d7508c5c1 ? 2026-03-11

## Ticket
Draft a no-code design document for integrating OpenClaw Pi engine into the Kayley codebase.

## Codebase Discoveries
- Design and planning docs live under `documents/`, with an index in `documents/README.md`.

## Gotchas & Bugs
- The `documents/features/` area is reserved for completed, production features; design docs should not be placed there unless the feature is already implemented.

## Approach That Worked
- Added a new design doc in `documents/` and linked it from the documentation index.

## What Future Opey Should Know
- Prefer the `documents/` root for architecture or integration designs, and update `documents/README.md` so the doc is discoverable.
