# Lessons Learned - 39ba1cc1-f182-4660-aeb9-c436256a2daf - 2026-03-10

## Ticket
Add a nightly cleanup job to purge server runtime logs and engineering ticket events older than one week.

## Codebase Discoveries
- The cron scheduler already owns server-side maintenance jobs; new action types belong in `server/scheduler/cronScheduler.ts`.
- Existing cron job creation SQL lives under `agents/**/migrations` and is intended to be run manually in Supabase.

## Gotchas & Bugs
- The correct table name is `engineering_ticket_events` (singular), not `engineering_tickets_events`.

## Approach That Worked
- Added a `runtime_log_cleanup` handler in the main cron scheduler and a matching SQL insert to register the nightly job.

## What Future Opey Should Know
- Use `created_at` for retention cleanup in `server_runtime_logs`; it is indexed and consistent with other tables.
