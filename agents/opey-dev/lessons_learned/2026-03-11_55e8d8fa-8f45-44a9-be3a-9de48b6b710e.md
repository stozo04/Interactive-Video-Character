# Lessons Learned - 55e8d8fa-8f45-44a9-be3a-9de48b6b710e - 2026-03-11

## Ticket
Implement Kayley autonomous health dashboard (kayley-pulse) with scheduled checks, snapshot file, and manual tool trigger.

## Codebase Discoveries
- Server background loops are started in server/index.ts alongside cron, calendar, and X mention heartbeats.
- Tool visibility uses server/services/ai/sseTypes.ts for display labels; new tools should be added there.

## Gotchas & Bugs
- The server codebase is ESM; use fileURLToPath(import.meta.url) instead of __dirname when resolving paths.

## Approach That Worked
- Implemented a server-side pulse service that polls health endpoints, writes pulse-config.json with a 50-run history, and exposed a kayley_pulse tool that can read or trigger checks on demand.

## What Future Opey Should Know
- Keep pulse logic server-side and let tools call into it via dynamic imports in executeMemoryTool.
- When adding a new tool, update aiSchema tool declarations and the SSE display name map together to keep UI visibility consistent.
